Part 2: Containers

Prerequisites

  • Install Docker Version 1.13 or higher.
  • Read Orientation or Part 1
  • Run docker and try a quick test run to make sure that everything is working fine: docker run hello-world

Introduction and Development Environment

A docker app is divided in 3 parts:

  • Stack
  • Services
  • Container

For example, if you are working on an app being written in java, the first thing to do would be to install the java runtime framework and the java SDK onto your machine. This creates a situation where the environment on your machine has to be in perfect order for your app to run as expected and same for the server that supports the application.

With Docker, this is not nescessary! You can just directly grab the portable java runtime as an image, absolutely no installation necessary.

These portable images are defined by Dockerfile

Defining an Image

A basic way to think about Dockerfile is to imgine it as a settings page. If your app neeeds access to particular resources like networking interfaces or certain disk drivers or packages you have to manually map ports to those specific things. However, once you're done with this step, you can except your docker app defined in this Dockerfile to run exactly the same everywhere.

All the packages needed by your app are simply mentioned in a file called requirements.txt which is referred to inside the Dockerfile. The file requirements.txt is placed in the same directory as Dockerfile.

Type the code in Dockerfile for installing packages: RUN pip install -r requirements.txt

This completes our app which as you can see is quite simple! Now you are ready to build your app. To build the app simply run this command docker build -t *appname*

Running the App

Run the app, mapping your machine's port 4000 to lets say containers port 80 using -p:

docker run -p 4000:80 *appname*

Congratulations! you just ran your first docker app!
